# Онкоскрининг — дашборд отчётности по КнСК

Веб-приложение на **Google Apps Script** для подготовки, просмотра и архивирования отчётов по онкоскринингу (КнСК) в разрезе медицинских организаций (МО). Данные загружаются из CSV или из архива в Google Таблице; дашборд строит KPI, графики, сигналы и сравнение периодов.

---

## Возможности

### Режим редактора (`Index.html`)

Полный цикл работы с отчётом:

| Функция | Описание |
|--------|----------|
| **Загрузка CSV** | Парсинг выгрузки с колонками МО, плана, факта, динамики, колоноскопии, ЗНО и др. |
| **Недельная динамика** | Ручной ввод недель (даты + значения), графики «Недельный план КнСК» и «Динамика КнСК» |
| **Текстовые блоки** | Редактор «Сделано» и «Планы» с простым форматированием (жирный, ссылка, список) |
| **Период отчёта** | Поле периода, отображается в шапке дашборда |
| **Архив** | Сохранение снимка отчёта (данные МО, недели, тексты, период) в лист «Архив» |
| **Загрузка из архива** | Выбор отчёта из списка и восстановление на дашборде |
| **Сравнение периодов** | Два архивных отчёта: ранний → поздний, дельты KPI и топ-5 МО |
| **Экспорт HTML** | Сохранение автономной HTML-копии дашборда на диск |
| **Презентация** | Полноэкранный режим без панелей управления |
| **Ссылка на viewer** | Кнопка «Веб-версия» (URL развёртывания) |

### Режим просмотра (`Viewer.html`)

Только чтение, без редактирования CSV и текстов:

- Выбор отчёта из архива (список подгружается при открытии)
- Тот же дашборд, что в редакторе после загрузки данных
- Сравнение двух архивных отчётов
- Презентация (кнопка на месте бывшей «Обновить список»)
- Открытие по URL: `?viewer=1` и опционально `&id=<ID отчёта>`

## Дашборд (что видит пользователь)

### Блок «Ключевые сигналы»

Автоматические заметки по текущей выборке МО:

- МО ниже порога **70%** годового плана КнСК
- Недельный прирост относительно плана **4 583** исследований/нед.
- Лидеры прироста колоноскопий (топ-3 МО)

При загрузке из архива — сравнение с **предыдущим** сохранённым отчётом (дельты в KPI).

### KPI (4 карточки)

1. **План на год** — % от **220 000** (`PLAN_YEAR` в `config.js`), факт/план
2. **КнСК с начала года** — суммарный факт и прирост за неделю
3. **Прошли колоноскопию** — количество и % от пациентов с отклонениями (КнСК+)
4. **Выявлено ЗНО** — 1–2 стадия

### Графики (Chart.js)

- Недельный план vs факт (по введённым неделям)
- Динамика КнСК по неделям
- **Топ-5 / антитоп-5** по % плана КнСК (горизонтальные бары, ось 70–170% для топа)
- **Топ-5 / антитоп-5** по охвату колоноскопией (% и абсолюты в подписи)
- **Охват колоноскопией** — все МО, горизонтальная прокрутка

### Сводная таблица по МО

- Сортировка по **% выполнения плана КнСК** (убывание)
- Цветовая шкала в колонке «%»
- **Поиск** по названию МО
- **Клик по строке** — боковая панель (drawer) с метриками МО и мини-графиком (ECharts)

### Сравнение двух архивных отчётов

- Слева в селекте — **ранний** период, справа — **поздний**
- Изменение считается как **поздний − ранний** (рост — зелёный, снижение — красный)
- KPI: факт КнСК, % годового плана, прирост за неделю, колоноскопия, ЗНО
- Два списка топ-5 МО: по приросту **факта КнСК** и по приросту **колоноскопий**
- Кнопка **×** в правом верхнем углу блока — скрыть панель

### Режим презентации

- Скрываются панели управления, сравнение, поиск по таблице
- Кнопка **«Выйти из презентации»** — в **левом верхнем** углу
- Закрытие также по **Esc**

## Backend (`Code.js`)

Привязка к **активной Google Таблице** проекта Apps Script.

| Функция | Назначение |
|---------|------------|
| `doGet(e)` | `Index` (редактор) или `Viewer` при `?viewer=1` |
| `saveReportToArchive(reportData)` | Запись в лист «Архив»: ID, дата, JSON, период |
| `getArchivedReportsList()` | Список `{ id, dateStr }` для выпадающих списков |
| `getArchivedReportById(id)` | Полный отчёт + нормализация `mosData` / `MOSData` |
| `include(filename)` | Подключение HTML-фрагментов в шаблоны |
| `getCurrentData()` | Чтение листа «Данные» (вспомогательно) |
| `migrateArchiveDates()` / `replaceBrokenArchive()` | Служебные операции в редакторе GAS |

**Лист «Архив»:** колонки `ID | Дата сохранения | Данные (JSON) | Период`.

### Как подключаются модули в GAS

В `Index.html` и `Viewer.html`:

```html
<?!= include('UiTokens'); ?>
<?!= include('UiPhase2'); ?>
<script>
(function () {
  var _cfg = <?!= clientConfigJson ?>;
  window.CONFIG = _cfg;
})();
</script>
<?!= include('DashboardPhase1'); ?>
<?!= include('DashboardPhase2'); ?>
```

**Важно:** плановые константы меняются только в `config.js`. В продакшене выполняются файлы `*.html` в корне, а не `src/`. После правок в `src/dashboard/` или `src/ui/` нужно синхронизировать:

```bash
# Стили
(printf '%s\n' '<style>'; cat src/ui/tokens.css; printf '%s\n' '</style>') > UiTokens.html
(printf '%s\n' '<style>'; cat src/ui/phase2.css; printf '%s\n' '</style>') > UiPhase2.html

# Скрипты дашборда
(printf '%s\n' '<script>'; cat src/dashboard/dashboardPhase1.js; printf '%s\n' '</script>') > DashboardPhase1.html
(printf '%s\n' '<script>'; cat src/dashboard/dashboardPhase2.js; printf '%s\n' '</script>') > DashboardPhase2.html

clasp push
```

Затем в Google Apps Script: **Развёртывание → Управление развёртываниями → Новая версия** (или обновить существующее).

## Константы расчётов

Все плановые значения задаются **в одном месте** — в начале файла `config.js`:

```javascript
var PLAN_YEAR = 220000;
var PLAN_WEEKLY = 4583;
var PLAN_THRESHOLD = 70;
```

| Параметр | Значение | Где используется |
|----------|----------|------------------|
| Годовой план КнСК (`PLAN_YEAR`) | **220 000** | KPI, сигналы, сравнение периодов |
| Недельный план (`PLAN_WEEKLY`) | **4 583** | Графики недельной динамики, сигналы |
| Порог сигнала «низкий план» (`PLAN_THRESHOLD`) | **70%** | Блок «Ключевые сигналы» |

После изменения `config.js` выполните `clasp push` и обновите развёртывание в Apps Script.

Значения из `config.js` передаются в браузер через `Code.js` → `getClientConfigJson()` → `clientConfigJson` в шаблонах `Index.html` / `Viewer.html`. Тег `<script src="config.js">` в GAS **не работает** — файл серверный, не отдаётся клиенту напрямую.

## Установка и первый запуск

1. Создайте или откройте Google Таблицу, привязанную к проекту Apps Script.
2. Склонируйте репозиторий и выполните `clasp login` / `clasp clone` (если проект уже в облаке — настройте `scriptId` в `.clasp.json`).
3. Синхронизируйте `src/` → `*.html` (команды выше) и выполните `clasp push`.
4. В редакторе Apps Script: **Развернуть → Новое развёртывание → Веб-приложение**.
   - Выполнять от имени: **владелец**
   - Доступ: по вашей политике (сейчас в манифесте: `ANYONE_ANONYMOUS` — см. раздел «Безопасность»)
5. Откройте URL развёртывания — откроется редактор.
6. Viewer: тот же URL + `?viewer=1` или `?viewer=1&id=5` для конкретного отчёта.

При первом сохранении в архив лист **«Архив»** создаётся автоматически.

## Формат CSV

Ожидаются колонки (или уже обработанный JSON из архива с полями `name`, `plan`, `fact`, …):

- Наименование МО  
- План на год  
- Общий итог (факт)  
- Динамика с прошлой недели  
- Прошли колоноскопию  
- Выявлено ЗНО C18-C21  
- Нет отклонений / Есть отклонения  

Числа могут содержать пробелы, знаки `+`/`-`, скобки с процентами — парсер это учитывает.

## Безопасность

В `appsscript.json` указано:

```json
"access": "ANYONE_ANONYMOUS"
```

Любой, у кого есть ссылка на развёртывание, может открыть приложение и читать архив через API. Для внутренней отчётности рекомендуется ограничить доступ (домен организации, авторизованные пользователи) и не публиковать URL в открытых источниках.

В `Index.html` может быть захардкожен URL «Веб-версии» — обновляйте его после каждого нового развёртывания.
